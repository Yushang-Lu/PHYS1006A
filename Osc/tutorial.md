# 示波器的原理和应用

## `osc.tex` 与 `drawer.py` 食用指南

`Osc/osc.tex` 是这次示波器实验报告的半成品，版式、表格和插图位置基本已经搭好，使用者主要需要根据自己的实验记录补全内容。（前三页内容未做精细排版）

通常还需要自己填写或检查的部分包括：

- 实验预习中的两个问答
- 前三部分实验表格中的原始数据
- 李萨如图形测频部分的频率记录
- 封面上的班级、学号、姓名、实验日期等信息
- 结论与分析中是否和你自己的数据一致

`Osc/drawer.py` 用来处理第二部分“用示波器测量信号发生器输出的方波信号频率”的数据，并自动生成报告里使用的拟合图 `diagram.png`。

脚本中目前已经写入了一组示例数据：

- `X`：信号发生器给出的频率，单位 `Hz`
- `Y`：示波器测得的频率，单位 `Hz`

如果你要处理自己的实验数据，只需要修改这两个数组。脚本内部已经做了检查：

- `X` 和 `Y` 的长度必须相同
- `X` 和 `Y` 都必须正好包含 10 个元素

修改完成后，可以在项目根目录运行：

```bash
python3 Osc/drawer.py
```

也可以先进入 `Osc` 目录再运行：

```bash
python3 drawer.py
```

脚本会完成这些事情：

- 使用最小二乘法拟合直线 `Y = aX + b`
- 在终端输出拟合方程
- 自动生成或覆盖 `Osc/diagram.png`

`osc.tex` 会直接引用这张 `diagram.png`，所以重新运行脚本后，再编译 LaTeX 就能把新图放进实验报告。

这个脚本依赖：

- `numpy`
- `matplotlib`

如果运行时报缺少模块，需要先在当前 Python 环境中安装它们。

除了 `diagram.png` 以外，`osc.tex` 还会直接使用下面三张图片：

- `graph1.JPG`
- `graph2.JPG`
- `graph3.JPG`

这三张图对应李萨如图形。如果你有自己的实验截图，可以直接替换同名文件；如果想改成别的文件名，也可以同步修改 `osc.tex` 里的引用。

建议的使用顺序是：

1. 先把实验原始数据填进 `osc.tex` 的各个表格。
2. 再把第二部分的 10 组 `X`、`Y` 数据写入 `drawer.py` 并生成 `diagram.png`。
3. 如有需要，替换 `graph1.JPG`、`graph2.JPG`、`graph3.JPG` 为你自己的李萨如图截图。
4. 最后编译 `osc.tex`，检查排版、图片和结论是否都正确。

由于 `osc.tex` 使用的是 `ctexart`，一般建议用 `xelatex` 编译，例如：

```bash
cd Osc
xelatex osc.tex
```

如果你的环境里缺少中文字体，编译时可能会报 `ctex` 或字体相关错误，这时需要先补齐字体，或者再调整 LaTeX 的字体配置。
